主流布局方式

固定布局（静态布局）

即传统Web设计，整个网页只有一个固定宽度，不会随着屏幕大小而变化，当屏幕过小时会显示滚动条来查阅被遮挡的部分。

弹性布局（流式布局）

页面元素的宽度采用百分比规则根据屏幕大小自动适配，大部分移动站点均采用此种布局。主要的问题表现在如果屏幕尺度跨度太大，那么在相对其原始设计而言过小或过大的屏幕上不能正常显示。

自适应布局

该布局的特点是分别为不同的屏幕分辨率定义布局方式。当布局方式切换时，页面元素发生对应变化。但是在每个布局方式中，页面元素不随窗口大小的调整而发生变化。

响应式布局

可以把响应式布局看做是弹性布局和自适应布局的融合。在响应式布局中，每个页面元素都采用弹性布局，同时根据屏幕宽度的不同，展示不同的显示方案。

响应式布局的基本原则

1. 移动优先

移动优先，它的意思是，先为移动端设计样式，然后再根据需求去优化更大屏幕的样式。换句话说，假如你把移动端样式当成网站的默认样式，且以后不用去优化它，一步到位。那就更省事了！

移动优先促使我们考虑哪些才是用户最迫切的需求，用户最想要的是什么。

从大到小，我们可能会忽略掉一些重要的信息。从小到大，我们能够保证重要的信息一直都在。移动优先，因为加法容易减法难。

2. 模块内容的可伸缩性

页面中的每个模块以及每个内容单元均采用弹性布局，根据屏幕宽度的不同而自动伸缩。

响应式设计是由著名网页设计师Ethan Marcotte在2010年5月提出的设计概念，随后席卷前端和设计领域，成为了如今网页设计的大趋势。正如同Ethan所说：“响应式网站设计提供了一种全新的选择，这种基于栅格布局和CSS3的流动性网页设计，可以让网页随着屏幕变化而响应。这是一种更为统一，更加全面的设计技巧，一种打破网页固有型态和限制的灵活设计方法。”

这种兼顾多屏幕多场景的灵活设计，这就是响应式设计令人着迷的所在。这一点和编程开发领域所追求的“一次编写，到处运行”有着异曲同工之妙。所以它在短短几年内迅速火起来，然后成为大趋势，可以说是意料之中，水到渠成。响应式设计的网页现在已经很多了，而相关的教程也是数不胜数。国外的设计博客Pixel77搜集了15篇优秀的响应式网页设计的教程，让喜欢钻研的你来学习。

响应式设计（以下称 RWD ）是一种设计思维方式，而不是某种特定的技术。我理解的响应式设计，就是让 Web 内容的呈现从手机到27寸 iMac 都保持最佳使用体验。一开始接触 RWD 的时候，感觉它关注的都是移动端，实际上大屏幕以及高分屏也属于 RWD 需要在设计或技术层面考量的要素。

自适应布局给了你更多设计的空间，因为你只用考虑几种不同的状态。而在响应式布局中你却得考虑上百种不同的状态。虽然绝大部分状态差异较小，但仍然也算做差异。它使得把握设计最终效果变得更难，同样让响应式布局更加的难以测试和预测。但同时说难，这也算是响应式布局美的所在。

响应式布局的设备响应

移动设备的物理像素（设备分辨率，最大内容展示像素）和逻辑像素（最佳视觉呈现，相对于物理像素存在一个缩放比例）。

IOS像素查询

依据分辨率划分设备：

1. 超小型设备（手机），分辨率小于768；

2. 小型设备（竖屏Pad），分辨率大于等于768；

3. 中型设备（横屏Pad），分辨率大于等于1024；

4. 大型设备（台式机、笔记本），分辨率在1280、1366、1440、1600、1920及以上设备；

移动设备采用逻辑像素进行视觉呈现的控制开关：

<meta name="viewport" content="width=device-width, initial-scale=1, minimum-scale=1, maximum-scale=1,user-scalable=no">

采用CSS3的媒体查询进行适配：

@media screen and (min-width: 768px){ }

适配的断点设置

以流行的响应式框架Bootstrap为例，它的断点分别是：（大）1200（中）992（小）768（超小）。

网格系统

将每个布局单元划分为12等分，布局列数可在1列、2列、3列、4列、6列中间自由切换，布局单元的宽度包含间距在内；

布局单元之间，允许随着屏幕尺寸的不同，而设置不同的间距；
